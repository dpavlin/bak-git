# Evolutions in bak-git: Speed, Security, and Time Travel

Recent updates to `bak-git` have transformed it from a simple backup script into a more robust, secure, and performant tool. Here’s a breakdown of the major improvements.

## 1. Performance: DSL-Friendly Backups
When working over slow DSL connections, every bit counts.
- **Rsync Optimization**: Reduced verbosity from `-avv` to `-av`, minimizing unnecessary network traffic and processing overhead.
- **WireGuard Integration**: By switching the primary route to `klin` over WireGuard (`192.168.2.1`), throughput was nearly doubled compared to standard local routes, reaching the physical limits of the DSL line (~14 Mbps).
- **SSH Route Bypassing**: Configured the server to bypass slow proxies for local connections, reducing SSH handshake latency by 10x (0.5s down to 0.05s).

## 2. Security: Hardened Whitelisting & Alerts
The server is now protected against unauthorized enrollments and legacy connections.
- **Dynamic Whitelisting**: Connections are only accepted from hostnames listed in `whitelist.txt`.
- **IP Verification**: The whitelist now supports mandatory IP mapping (e.g., `hostname 192.168.2.40`). If a known host connects from an unexpected IP, it is denied.
- **Automated Alerts**: Any denied connection triggers an immediate email alert to the administrator. The email includes a pre-formatted snippet for `whitelist.txt`, making it easy to authorize new machines while staying informed of every attempt.

## 3. New Features: Relative Listing & Original Timestamps
- **`bak ls-files`**: A new command that mirrors `git ls-files`, but provides output relative to the user's current working directory, making it intuitive to use within subdirectories.
- **Timestamp Preservation**: `bak commit` now extracts the original file modification time (`mtime`) and uses it as the Git commit date. This means your backup history reflects when the work was actually done, not just when the backup was run.
- **FUSE /mnt/host Support**: Improved path parsing automatically detects and translates paths starting with `/mnt/host/` to the correct host in the backup repository, enabling seamless backups from remote mount points.

## 4. Reliability: The Test Runner
A new `test.sh` suite ensures that no new feature breaks existing functionality.
- **Isolated Testing**: Uses `mktemp` and local Git repos to test 100% of the logic without touching production data.
- **18 Automated Tests**: Covers everything from basic `add/commit` to complex edge cases like single-character paths (e.g., `bak diff --stat /`) and multi-word commit messages.

## 5. A Blast from the Past: The PROJECT_X Project
While analyzing logs to build the whitelist, we uncovered historical author data from the **PROJECT_X project (circa 2011-2012)**. We successfully separated 70 legacy hosts into a `legacy_hosts.txt` file, preserving the history of that infrastructure (including n2n community `p2p_net` on `192.168.3.x`) while keeping our current whitelist lean and active.
